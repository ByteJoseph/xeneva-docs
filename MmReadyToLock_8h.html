Detailed Description

MM Ready To Lock protocol introduced in the PI 1.5 specification.

This protocol is a mandatory protocol published by the MM Foundation code when the system is preparing to lock certain resources and interfaces in anticipation of the invocation of 3rd party extensible modules. This protocol is an MM counterpart of the DXE MM Ready to Lock Protocol. This protocol prorogates resource locking notification into MM environment. This protocol is installed after installation of the MM End of DXE Protocol.
